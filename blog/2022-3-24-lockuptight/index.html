Locking it Up Tight with Coreboot or Libreboot

First published Mar 24, 2022 at 3:00 AM

Last week, I got my hands on a couple of Core 2 Duo MacBooks (MacBook2,1) from 2007. I cleaned them up, gave them new thermal paste, and Librebooted them, one with SeaBIOS as the primary payload and the other with GRUB.

With a system that has GRUB as part of the init firmware, I wanted to try installing Linux without installing a bootloader, and I realized it could be more secure this way.

Without GRUB in Coreboot

NAME      FSTYPE      MOUNTPOINTS
sda
|-sda1    vfat        /boot
`-sda2    crypto_LUKS
  `-crypt btrfs       /home
                      /

Normally when a computer boots, it has to load a bootloader from the hard disk and execute it to boot into the operating system. The bootloader in the hard disk must be unencrypted and stored in a specific way for it to work.

Most Linux installations with full-disk encryption have the bootloader, kernel, and initramfs stored unencrypted in order to boot. These are three points that can be infected with malware without decrypting anything.

The advantage of using GRUB with Coreboot

With Coreboot and Libreboot, the GRUB bootloader can be stored as part of the initialization firmware, and it can boot into Linux without having to load a bootloader from the hard disk it's installed on.

With GRUB also being able to access LUKS encrypted systems, this means that it's possible to encrypt every last byte of the hard disk (other than the LUKS header) without any unencrypted bootloaders or boot filesystems. This makes the system more secure since there is nothing on the disk to externally infect with malware.
